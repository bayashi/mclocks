* 🔄 محوّل بين وقت Epoch والتاريخ والوقت

## 📦 التحميل

من https://github.com/bayashi/mclocks/releases

لنظام Windows، يمكنك الحصول على ملف المثبّت `.msi` أو الملف التنفيذي `.exe`.

## ⚙️ config.json

يتيح لك ملف `config.json` تكوين الساعات وفقاً لتفضيلاتك.

يجب أن يكون ملف `config.json` موجوداً في الأدلة التالية:

* Windows: `C:\Users\{USER}\AppData\Roaming\com.bayashi.mclocks\`
* Mac: `/Users/{USER}/Library/Application Support/com.bayashi.mclocks/`

<!-- * Linux: `/home/{USER}/.config/com.bayashi.mclocks/` -->

عند بدء `mclocks`، اضغط `Ctrl + o` لتحرير ملف `config.json` الخاص بك.

مُنشئ `config.json` عبر الويب: https://bayashi.github.io/mclocks/mclocks-config-generator/

### مثال config.json للساعة

يجب أن يكون ملف `config.json` بتنسيق JSON (متوافق مع JSONC)، كما هو موضح أدناه.

    {
      "clocks": [
        { "name": "UTC", "timezone": "UTC" }
      ],
      "format": "MM-DD ddd HH:mm",
      "locale": "en",
      "color": "#fff",
      "font": "Courier, monospace",
      "size": 14,
      "margin": "1.65em",
      "forefront": false
    }

الأقسام التالية تشرح الحقول التي يمكنك تعيينها في `config.json`.

#### clocks

حقل `clocks` هو مصفوفة من الكائنات، كل منها يحتوي على خصائص `name` و `timezone`. يجب أن يكون كلاهما نصاً. القيمة الافتراضية لكليهما هي `UTC`.

* `name` هو التسمية التي ستُعرض للساعة.
* لاختيار المناطق الزمنية، يرجى الرجوع إلى هذه [قائمة المناطق الزمنية](https://en.wikipedia.org/wiki/List_of_tz_database_time_zones).

فيما يلي مثال على مصفوفة `clocks` لثلاث مناطق زمنية.

    {
      "clocks": [
        { "name": "Tokyo", "timezone": "Asia/Tokyo" },
        { "name": "UTC", "timezone": "UTC" },
        { "name": "SF", "timezone": "America/Los_Angeles" }
      ],
      "format": "MM-DD ddd HH:mm",
      ...snip...
    }

#### format

نص: `MM-DD ddd HH:mm`

حقل `format` هو سلسلة تنسيق التاريخ والوقت المستخدمة لعرض الساعة. لإنشاء تنسيق مخصص، يرجى الرجوع إلى [دليل التنسيق هذا](https://momentjs.com/docs/#/parsing/string-format/).

#### format2

نص: `""`

حقل `format2` مماثل لـ `format`. يتم التبديل بينهما بمفتاح `Ctrl + f`. حقل `format2` اختياري.

#### locale

نص: `en`

حقل `locale` يحدد إعدادات اللغة لعرض التاريخ والوقت. يمكنك العثور على [قائمة اللغات المدعومة هنا](https://github.com/kawanet/cdate-locale/blob/main/locales.yml).

#### color

نص: `#fff`

حقل `color` يحدد لون نص التاريخ والوقت. يمكنك استخدام الألوان المسماة، أو قيم RGB الست عشرية، أو قيم RGB (مثل `RGB(255, 0, 0)`)، أو أي قيمة لون CSS صالحة.

#### font

نص: `Courier, monospace`

`font` هو اسم الخط لعرض التاريخ والوقت. يجب أن يكون خطاً أحادي المسافة. إذا قمت بتعيين خط متغير العرض، فقد يكون لتطبيق mclocks تأثير اهتزاز غير مرغوب فيه.

#### size

رقم | نص: 14

`size` هو حجم الحرف للتاريخ والوقت، بالبكسل. يمكن أيضاً تحديده كنص يتضمن وحدة (مثل `"125%"`، `"1.5em"`).

#### margin

نص: `1.65em`

حقل `margin` يحدد المسافة بين الساعات.

#### forefront

منطقي: `false`

إذا تم تعيين حقل `forefront` على `true`، فسيُعرض تطبيق mclocks دائماً فوق نوافذ التطبيقات الأخرى.

## ⏳ ساعة العد التنازلي

من خلال إعداد تكوين `clock` كما هو موضح أدناه، سيُعرض كساعة عد تنازلي إلى تاريخ ووقت `target` المحدد.

	"clocks": [
		{
			"countdown": "WAC Tokyo D-%D %h:%m:%s",
			"target": "2025-09-13",
			"timezone": "Asia/Tokyo"
		}
	],

ساعة العد التنازلي أعلاه ستُعرض كالتالي:

    WAC Tokyo D-159 12:34:56

تشير إلى أنه تبقى 159 يوماً و12 ساعة و34 دقيقة و56 ثانية حتى 13 سبتمبر 2025.

### رموز تنسيق العد التنازلي

يقبل نص حقل `countdown` رموز القالب التالية:

* `%TG`: سلسلة التاريخ والوقت المستهدف
* `%D`: عدد الأيام المتبقية حتى التاريخ والوقت المستهدف
* `%H`: الوقت المتبقي بالساعات حتى التاريخ والوقت المستهدف
* `%h`: جزء الساعة (hh) من الوقت المتبقي (hh:mm:ss)
* `%M`: الوقت المتبقي بالدقائق حتى التاريخ والوقت المستهدف
* `%m`: جزء الدقيقة (mm) من الوقت المتبقي (hh:mm:ss)
* `%S`: الوقت المتبقي بالثواني حتى التاريخ والوقت المستهدف
* `%s`: جزء الثانية (ss) من الوقت المتبقي (hh:mm:ss)

## ⏱️ مؤقت بسيط

![simple timer](https://raw.githubusercontent.com/bayashi/mclocks/main/screenshot/mclocks-screenshot-timer.png)

انقر على نافذة تطبيق `mclocks`، ثم اضغط مفتاح `Ctrl + 1` لبدء مؤقت لمدة دقيقة واحدة. اضغط مفتاح `Ctrl + Alt + 1` لبدء مؤقت لمدة 10 دقائق. مفاتيح الأرقام الأخرى تعمل بنفس الطريقة. يمكن تشغيل ما يصل إلى 5 مؤقتات في وقت واحد.

`Ctrl + p` لإيقاف / استئناف المؤقتات.

`Ctrl + 0` لحذف أقدم مؤقت. `Ctrl + Alt + 0` لحذف أحدث مؤقت.

🔔 ملاحظة: ساعة العد التنازلي والمؤقت البسيط يرسلان إشعاراً افتراضياً عند اكتمال المؤقت. إذا لم تكن بحاجة إلى الإشعارات، قم بتعيين `withoutNotification: true` في `config.json`.

## 🔄 التحويل بين التاريخ والوقت ووقت Epoch

انقر على نافذة تطبيق `mclocks`، ثم الصق تاريخاً ووقتاً أو وقت Epoch، وسيظهر مربع حوار لعرض نتائج التحويل. ويمكنك نسخ النتائج إلى الحافظة. إذا لم ترغب في النسخ، اضغط `[No]` لإغلاق مربع الحوار فقط.

عند اللصق باستخدام `Ctrl + v`، تُعامل القيمة (وقت Epoch) كثوانٍ. إذا استخدمت `Ctrl + Alt + v`، تُعامل كمللي ثانية، ومع `Ctrl + Alt + Shift + V` كميكرو ثانية، ومع `Ctrl + Alt + Shift + N + V` كنانو ثانية ويتم التحويل وفقاً لذلك.

## 🔀 ميزة تحويل النصوص

انقر على نافذة تطبيق `mclocks`، ثم استخدم اختصارات لوحة المفاتيح التالية لمعالجة نص الحافظة وفتحه في المحرر:

* `Ctrl + i`: يحيط كل سطر من نص الحافظة بعلامات اقتباس مزدوجة ويضيف فاصلة في النهاية (باستثناء السطر الأخير)
* `Ctrl + Shift + i`: يضيف فاصلة في نهاية كل سطر (بدون علامات اقتباس) لشرط IN لقوائم INT (باستثناء السطر الأخير)

تُحفظ الأسطر الفارغة كما هي في جميع العمليات.

(ميزة تحويل النصوص هذه لا علاقة لها بالساعات أو الوقت، لكن مطوري البرمجيات قد يجدونها مفيدة! 😊)

## 📝 ملاحظة لاصقة

![sticky-note](https://raw.githubusercontent.com/bayashi/mclocks/main/screenshot/mclocks-screenshot-sticky-note.png)

انقر على نافذة تطبيق `mclocks`، ثم اضغط `Ctrl + s` لإنشاء ملاحظة لاصقة من نص الحافظة. ستُفتح نافذة عائمة صغيرة بمحتوى الحافظة.

كل ملاحظة لاصقة تحتوي على:

* **زر التبديل** (`▸` / `▾`): توسيع أو طي الملاحظة. في وضع الطي يُعرض سطر واحد فقط.
* **زر النسخ** (`⧉`): نسخ نص الملاحظة إلى الحافظة.
* **زر المقدمة** (`⊤` / `⊥`): تبديل ما إذا كانت الملاحظة تبقى فوق النوافذ الأخرى. يُحفظ هذا الإعداد لكل ملاحظة لاصقة.
* **زر الإغلاق** (`✖`): حذف الملاحظة اللاصقة وإغلاق نافذتها.
* **منطقة النص**: تحرير محتوى الملاحظة بحرية. تُحفظ التغييرات تلقائياً.
* **مقبض تغيير الحجم**: اسحب الزاوية السفلية اليمنى لتغيير حجم الملاحظة عند توسيعها.

ترث الملاحظات اللاصقة إعدادات `font` و `size` و `color` و `forefront` من `config.json`. يمكن تجاوز إعداد المقدمة لكل ملاحظة لاصقة باستخدام زر المقدمة؛ إذا لم يتم التجاوز، تُستخدم القيمة من `config.json`. يتم الاحتفاظ بموقعها وحجمها وحالة الفتح/الإغلاق وتجاوز المقدمة، وتُستعاد جميع الملاحظات تلقائياً عند إعادة تشغيل `mclocks`.

الحد الأقصى لحجم النص لكل ملاحظة لاصقة هو 128 كيلوبايت.

## 🌐 خادم الويب

يبدأ `mclocks` دائماً خادم ويب محلي مدمجاً عند التشغيل. إذا قمت بتكوين حقل `web` في `config.json`، يمكنه أيضاً تقديم الملفات الثابتة من دليلك:

    {
      "web": {
        "root": "/path/to/your/webroot",
        "dump": true,
        "slow": true,
        "status": true,
        "content": {
          "markdown": {
            "allowRawHTML": false,
            "openExternalLinkInNewTab": true
          }
        },
        "editor": {
          "reposDir": "/path/to/your/repos"
        }
      }
    }

* `root`: مسار الدليل الذي يحتوي على الملفات المراد تقديمها (مطلوب فقط عند استخدام استضافة الملفات الثابتة)
* `port`: رقم المنفذ المفضل لخادم الويب الرئيسي (`>=2000`، الافتراضي: `3030`). إذا كان مشغولاً، يبحث mclocks عن منفذ متاح بتقليل الرقم (`-1`).
* `openBrowserAtStart`: إذا كان `true`، يفتح عنوان URL لخادم الويب في المتصفح الافتراضي عند بدء `mclocks` (الافتراضي: `false`)
* `dump`: إذا كان `true`، يُفعّل نقطة النهاية `/dump` التي تُعيد تفاصيل الطلب كـ JSON (الافتراضي: `false`)
* `slow`: إذا كان `true`، يُفعّل نقطة النهاية `/slow` التي تؤخر الاستجابة (الافتراضي: `false`)
* `status`: إذا كان `true`، يُفعّل نقطة النهاية `/status/{code}` التي تُعيد رموز حالة HTTP عشوائية (الافتراضي: `false`)
* `content.markdown.allowRawHTML`: إذا كان `true`، يسمح بـ HTML الخام في عرض Markdown؛ إذا كان `false`، يُهرب HTML الخام كنص (الافتراضي: `false`)
* `content.markdown.openExternalLinkInNewTab`: تفتح روابط Markdown الخارجية في علامة تبويب جديدة والداخلية في نفس العلامة؛ إذا كان `false`، تفتح جميع روابط Markdown في نفس العلامة (الافتراضي: `true`)
* `content.markdown.enablePreviewApi`: إذا كان `true`، يُفعّل `POST /preview` لمعاينة Markdown من سطر الأوامر في المتصفح (الافتراضي: `false`).
* `editor`: إذا تم تعيينه ويحتوي على `reposDir`، يُفعّل نقطة النهاية `/editor` لفتح الملفات المحلية في المحرر من روابط GitHub في المتصفح (الافتراضي: غير معيّن)

### عارض محتوى بالسحب والإفلات

بالإضافة إلى استضافة الملفات الثابتة، يدعم mclocks سير عمل عارض محتوى بالسحب والإفلات:

* أفلت دليلاً على نافذة الساعة لفتحه في عارض الويب عبر URL محلي مؤقت.
* أفلت ملفاً واحداً لفتحه في عارض الويب عندما يدعمه عارض الملفات المؤقت.
* عناوين URL المؤقتة المُنشأة محلية فقط وتُستبعد عند الخروج من mclocks.

#### وضع المحتوى

يدعم عارض الويب خيارات استعلام `mode` مثل `content` و`raw` و`source`.

* `content` (الافتراضي): يقدّم الملف بنوع المحتوى المكتشف ليعرضه المتصفح عادياً عند الإمكان.
* `raw`: يُرجع الملفات غير الثنائية كـ `text/plain` لعرض النص الخام دون عرض المتصفح.
* `source`: يفتح تخطيط عارض المصدر مع ملخص/شريط جانبي للصيغ المدعومة، مع إمكانية فحص نص عادي بأمان للملفات النصية غير المدعومة.

**Markdown** **يكتشف التغييرات تلقائيًا ويعكسها في المتصفح في الوقت الفعلي** (عرض مُصيَّر في وضع **`source`**).

### نقطة النهاية /dump

عند تعيين `dump: true` في تكوين `web`، يوفر خادم الويب نقطة النهاية `/dump` التي تُعيد تفاصيل الطلب بتنسيق JSON.

تستجيب نقطة النهاية بكائن JSON يحتوي على:
* `method`: طريقة HTTP (مثل "GET"، "POST")
* `path`: مسار الطلب بعد `/dump/` (مثل "/test" لـ `/dump/test`)
* `query`: معلمات الاستعلام كمصفوفة من كائنات مفتاح-قيمة (مثل `[{"key1": "value1"}, {"key2": "value2"}]`)
* `headers`: رؤوس الطلب كمصفوفة من كائنات مفتاح-قيمة (مثل `[{"Content-Type": "application/json"}]`)
* `body`: جسم الطلب كنص (إذا كان موجوداً)
* `parsed_body`: كائن JSON مُحلَّل إذا كان Content-Type يشير إلى JSON، أو سلسلة رسالة خطأ إذا فشل التحليل

يمكنك الوصول إلى نقطة النهاية dump على `http://127.0.0.1:3030/dump` أو أي مسار تحت `/dump/` (مثل `/dump/test?key=value`).

### نقطة النهاية /slow

عند تعيين `slow: true` في تكوين `web`، يوفر خادم الويب نقطة النهاية `/slow` التي تؤخر الاستجابة قبل إعادة 200 OK.

نقطة النهاية متاحة عبر أي طريقة HTTP (GET، POST، إلخ) وتدعم المسارات التالية:

* `/slow`: تنتظر 30 ثانية (افتراضياً) وتُعيد 200 OK
* `/slow/120`: تنتظر 120 ثانية (أو أي عدد محدد من الثواني) وتُعيد 200 OK

القيمة القصوى المسموح بها هي 901 ثانية (15 دقيقة + ثانية واحدة). الطلبات التي تتجاوز هذا الحد تُعيد خطأ 400 Bad Request.

نقطة النهاية هذه مفيدة لاختبار سلوك المهلة الزمنية ومعالجة الاتصالات أو محاكاة ظروف الشبكة البطيئة.

إذا تم توفير معلمة ثوانٍ غير صالحة (مثل `/slow/abc`)، تُعيد نقطة النهاية خطأ 400 Bad Request.

### نقطة النهاية /status

عند تعيين `status: true` في تكوين `web`، يوفر خادم الويب نقطة النهاية `/status/{code}` التي تُعيد رموز حالة HTTP عشوائية محددة في معايير RFC (100-599).

تُعيد نقطة النهاية استجابة نص عادي مع رمز الحالة والعبارة المقابلة له، مع رؤوس مناسبة كما هو مطلوب بمواصفات HTTP.

**أمثلة:**
* `http://127.0.0.1:3030/status/200` - يُعيد 200 OK
* `http://127.0.0.1:3030/status/404` - يُعيد 404 Not Found
* `http://127.0.0.1:3030/status/500` - يُعيد 500 Internal Server Error
* `http://127.0.0.1:3030/status/418` - يُعيد 418 I'm a teapot (مع رسالة خاصة)
* `http://127.0.0.1:3030/status/301` - يُعيد 301 Moved Permanently (مع رأس Location)

**رؤوس خاصة بالحالة:**

تضيف نقطة النهاية تلقائياً رؤوساً مناسبة لرموز حالة محددة:

* **3xx إعادة توجيه** (301, 302, 303, 305, 307, 308): تضيف رأس `Location`
* **401 Unauthorized**: تضيف رأس `WWW-Authenticate`
* **405 Method Not Allowed**: تضيف رأس `Allow`
* **407 Proxy Authentication Required**: تضيف رأس `Proxy-Authenticate`
* **416 Range Not Satisfiable**: تضيف رأس `Content-Range`
* **426 Upgrade Required**: تضيف رأس `Upgrade`
* **429 Too Many Requests**: تضيف رأس `Retry-After` (60 ثانية)
* **503 Service Unavailable**: تضيف رأس `Retry-After` (60 ثانية)
* **511 Network Authentication Required**: تضيف رأس `WWW-Authenticate`

**معالجة جسم الاستجابة:**

* **204 No Content** و **304 Not Modified**: تُعيد جسم استجابة فارغ (وفقاً لمواصفات HTTP)
* **418 I'm a teapot**: تُعيد الرسالة الخاصة "I'm a teapot" بدلاً من عبارة الحالة القياسية
* **جميع رموز الحالة الأخرى**: تُعيد نصاً عادياً بتنسيق `{code} {phrase}` (مثل "404 Not Found")

نقطة النهاية هذه مفيدة لاختبار كيفية تعامل تطبيقاتك مع رموز حالة HTTP المختلفة ومعالجة الأخطاء وإعادة التوجيهات ومتطلبات المصادقة وسيناريوهات تحديد المعدل.

### نقطة النهاية /editor

عند تعيين `web.editor.reposDir` في ملف التكوين، يوفر خادم الويب نقطة النهاية `/editor` التي تتيح لك فتح الملفات المحلية في محررك مباشرة من روابط GitHub في المتصفح.

**التكوين:**

أضف ما يلي إلى تكوين `web` الخاص بـ `editor`:

    {
      "web": {
        "root": "/path/to/your/webroot",
        "editor": {
          "reposDir": "~/repos",
          "includeHost": false,
          "command": "code",
          "args": ["-g", "{file}:{line}"]
        }
      }
    }

* `reposDir`: مسار دليل المستودعات المحلية. يدعم `~` لتوسيع الدليل الرئيسي (مثل `"~/repos"` على macOS أو `"C:/Users/username/repos"` على Windows). يجب أن يكون هذا الدليل موجوداً.
* `includeHost`: إذا كان `true`، يتضمن حل المسار المحلي المضيف الأصلي كدليل (مثل `{reposDir}/{host}/{owner}/{repo}/...`). إذا كان `false`، يُحل إلى `{reposDir}/{owner}/{repo}/...` (الافتراضي: `false`).
* `command`: اسم الأمر أو مسار الملف التنفيذي لمحررك (الافتراضي: `code`)
* `args`: مصفوفة قالب الوسائط. استخدم العناصر النائبة `{file}` و `{line}`. إذا لم يكن `#L...` موجوداً في URL، يستخدم `{line}` القيمة 1.

**كيف يعمل:**

1. عند الوصول إلى URL ملف GitHub عبر نقطة النهاية `/editor`، يُحوّل مسار GitHub إلى مسار ملف محلي
2. يُبنى مسار الملف المحلي كالتالي: `{reposDir}/{owner}/{repository_name}/{file_path}`
3. إذا كان الملف موجوداً، يفتحه في محررك عند رقم السطر المحدد باستخدام الأمر والوسائط المكوّنة (الافتراضي: `code -g {local_file_path}:{line_number}`)
4. إذا لم يكن الملف موجوداً، تُعرض صفحة خطأ مع رابط لاستنساخ المستودع

**إشارة مرجعية:**

أنشئ إشارة مرجعية لفتح ملفات GitHub بسرعة في محررك المحلي. استبدل `3030` برقم المنفذ المكوّن لديك:

```javascript
javascript:(function(){var u=new URL(document.location.href);open('http://127.0.0.1:3030/editor/'+u.host+u.pathname+u.hash,'_blank');})()
```

**دعم رقم السطر:**

يمكنك تحديد رقم السطر باستخدام جزء الهاش في URL:
* `https://github.com/username/repo/blob/main/file.rs#L123` → يفتح عند السطر 123

**معالجة الأخطاء:**

* إذا لم يكن الملف موجوداً محلياً، تبقى علامة التبويب مفتوحة وتعرض رسالة خطأ مع رابط لاستنساخ المستودع من GitHub
* إذا تم فتح الملف بنجاح، تُغلق علامة التبويب تلقائياً
* إذا لم يتم تكوين `web.editor.reposDir` أو لم يكن موجوداً، لن يتم تفعيل نقطة النهاية `/editor` (وستحصل على 404)

**مثال:**

1. أنت تتصفح ملفاً على GitHub: `https://github.com/bayashi/mclocks/blob/main/src/app.js#L42`
2. انقر على الإشارة المرجعية أو انتقل يدوياً إلى: `http://127.0.0.1:3030/editor/bayashi/mclocks/blob/main/src/app.js#L42`
3. إذا كان `~/repos/mclocks/src/app.js` موجوداً محلياً، يفتحه VS Code عند السطر 42
4. إذا لم يكن الملف موجوداً، تُعرض صفحة خطأ مع رابط إلى `https://github.com/bayashi/mclocks` للاستنساخ

## 🧠 خادم MCP لـ mclocks

يتضمن `mclocks` خادم MCP (بروتوكول سياق النموذج) الذي يُمكّن مساعدي الذكاء الاصطناعي مثل [Cursor](https://www.cursor.com/) و [Claude Desktop](https://claude.ai/download) من الإجابة على "كم الساعة الآن؟" عبر مناطق زمنية متعددة، والتحويل بين تنسيقات التاريخ والوقت وطوابع Epoch الزمنية. يستخدم خادم MCP تلقائياً ملف `config.json` الخاص بـ mclocks، لذا تنعكس المناطق الزمنية التي كوّنتها في mclocks في استجابات الذكاء الاصطناعي.

### المتطلبات الأساسية

* [Node.js](https://nodejs.org/) (الإصدار 18 أو أحدث)

إذا لم يكن لديك Node.js، قم بتثبيته من الموقع الرسمي.

### الإعداد

أضف JSON التالي إلى ملف تكوين MCP الخاص بك:

* **Cursor**: `.cursor/mcp.json` في جذر مشروعك، أو عالمياً `~/.cursor/mcp.json`
* **Claude Desktop** (`claude_desktop_config.json`):
  * Windows: `%APPDATA%\Claude\claude_desktop_config.json`
  * macOS: `~/Library/Application Support/Claude/claude_desktop_config.json`
  * Linux: `~/.config/Claude/claude_desktop_config.json`

```json
{
  "mcpServers": {
    "mclocks-datetime-util": {
      "command": "npx",
      "args": ["-y", "mclocks-datetime-util"]
    }
  }
}
```

بعد الحفظ، أعد تشغيل التطبيق. سيتم تنزيل خادم MCP وبدء تشغيله تلقائياً. ستتوفر الأدوات التالية:

* **`current-time`** - الحصول على الوقت الحالي في مناطقك الزمنية المكوّنة
* **`local-time`** - الحصول على الوقت المحلي الحالي في منطقة المستخدم الزمنية (من تكوين `convtz` أو الإعداد الافتراضي للنظام)
* **`convert-time`** - تحويل سلسلة تاريخ ووقت أو طابع Epoch زمني إلى مناطق زمنية متعددة
* **`next-weekday`** - إيجاد تاريخ الحدوث التالي ليوم محدد من الأسبوع
* **`date-to-weekday`** - الحصول على يوم الأسبوع لتاريخ محدد
* **`days-until`** - حساب عدد الأيام من اليوم حتى تاريخ محدد
* **`days-between`** - حساب عدد الأيام بين تاريخين
* **`date-offset`** - حساب التاريخ قبل أو بعد N يوم من تاريخ محدد

### كيف يعمل مع تكوين mclocks

يقرأ خادم MCP تلقائياً ملف `config.json` الخاص بـ mclocks ويستخدم:

* **`clocks`** - المناطق الزمنية المحددة في ساعاتك تُستخدم كأهداف تحويل افتراضية
* **`convtz`** - يُستخدم كمنطقة زمنية مصدر افتراضية عند تحويل سلاسل التاريخ والوقت بدون معلومات المنطقة الزمنية
* **`usetz`** - يُفعّل التحويل الصارم للمناطق الزمنية لإزاحات UTC الدقيقة تاريخياً (مثلاً، كان JST +09:18 قبل عام 1888). اضبطه على `true` عندما تحتاج إلى تحويل تواريخ وأوقات تاريخية بدقة

إذا لم يُعثر على `config.json`، يعود الخادم إلى مجموعة مدمجة من المناطق الزمنية الشائعة (UTC، America/New_York، America/Los_Angeles، Europe/London، Europe/Berlin، Asia/Tokyo، Asia/Shanghai، Asia/Kolkata، Australia/Sydney).

### متغيرات البيئة

إذا كنت تريد تجاوز إعدادات `config.json`، أو إذا لم يكن لديك `config.json` أصلاً، يمكنك تعيين متغيرات البيئة في تكوين MCP الخاص بك. متغيرات البيئة لها الأولوية على القيم في `config.json`.

| المتغير | الوصف | الافتراضي |
|---|---|---|
| `MCLOCKS_CONFIG_PATH` | مسار `config.json`. غير مطلوب في معظم الحالات، حيث يكتشف الخادم الموقع تلقائياً. | اكتشاف تلقائي |
| `MCLOCKS_LOCALE` | اللغة لتنسيق أسماء أيام الأسبوع وغيرها (مثل `ja`، `pt`، `de`) | `en` |
| `MCLOCKS_CONVTZ` | المنطقة الزمنية المصدر الافتراضية لتفسير سلاسل التاريخ والوقت بدون معلومات المنطقة الزمنية (مثل `Asia/Tokyo`) | *(لا شيء)* |
| `MCLOCKS_USETZ` | اضبط على `true` لتفعيل التحويل الصارم للمناطق الزمنية | `false` |

مثال:

```json
{
  "mcpServers": {
    "mclocks-datetime-util": {
      "command": "npx",
      "args": ["-y", "mclocks-datetime-util"],
      "env": {
        "MCLOCKS_LOCALE": "ja",
        "MCLOCKS_CONVTZ": "Asia/Tokyo"
      }
    }
  }
}
```

### مثال على الاستخدام

بمجرد التكوين، يمكنك سؤال مساعد الذكاء الاصطناعي أشياء مثل:

* "كم الساعة الآن؟" - يُعيد الوقت الحالي في جميع مناطقك الزمنية المكوّنة في mclocks
* "كم الساعة في جاكرتا؟" - يُعيد الوقت الحالي في منطقة زمنية محددة
* "حوّل epoch 1705312200 إلى تاريخ ووقت"
* "حوّل 2024-01-15T10:30:00Z إلى Asia/Tokyo"
* "ما تاريخ الجمعة القادمة؟"
* "ما هو يوم 25 ديسمبر 2026 من الأسبوع؟"
* "كم يوماً متبقياً حتى عيد الميلاد؟"
* "كم يوماً بين 1 يناير 2026 و 31 ديسمبر 2026؟"
* "ما التاريخ بعد 90 يوماً من 1 أبريل 2026؟"
